## Slide 7 - Demo 2: GitHub Copilot in VS Code

### How a task is executed live

- Insight: the biggest gains come from a governed loop, not from one-shot prompting
- Evidence: practitioner and vendor sources converge on harness plus evals plus human review
- Caveat: unreviewed autonomous output can increase rework and hidden errors
- RIVM implication: standardize one loop: prompt -> draft -> human review -> test -> release

## Slide 10 - Summary

### Key takeaways from this session

- Start with one concrete workflow and define baseline metrics first
- Use Copilot to accelerate drafting, but keep human review and fact-checking mandatory
- Scale only when quality is stable and responsibilities are explicit
- Treat AI as a controlled productivity tool, not as autonomous decision-maker

Practical conclusion:

- Pilot, measure, validate, and then expand in a controlled way.
