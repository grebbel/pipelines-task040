# Automating deployment pipelines with GitHub Copilot in Visual Studio Code

## Objectives

Presentation and Demonstration on applying Visual Code and GitHub CoPilot for projects.

* Understand what are Visual Studio Code and Github Copilot
* Write a Product Requirement Document
* Identify opportunities for similar implementations in other RIVM projects

## Duration

The times are approximate and flexible

5 minutes HTML slides

5 minutes demo elearning

5 minutes demo Visual Studio Code

5 minutes HTML slides

## Content

### Slides

1. Welcome slide
2. What's in it for you?
3. A lot to say about AI: Privacy, big tech, Environment pressure, unreliable. There are also some good developments, such as the RIVM Chatbox, the European initiative Mistral. The disadvantages are all very valid arguments. However, that is not the focus for this presentation.
4. So what's in it for us? To start with, during my PhD research, I used the SPSS software. It was paid by the University. In my room there was this uber-geek, telling me to use R. I did not have time for it. After my PhD, when I had to do statistics, I could not affort to buy SPSS. Then I remembered this R-Geek. I took a course and a new world opened up for me. The possibilities were limitless. Today, SPSS is almost non-existend anymore. Back to the RIVM. To create e-learnings, we bought Articulate Software. It's ridiculously expensive. And after a while, there was no out-of-the-box possibilites. So I tried, as a pilot, this alternative way of building an e-learning. It's been a learning process, but again, i now see so many applications and possibilities. This I want to share with you.

### Demo e-learning

During this demo I will go trough the e-learning in `11_MODULE_1`. In particular module chapter `Laboratory diagnosis`, with the interactive clinical case. The course progress and scores are recorded for the Moodle lms.

### Demo Github CoPilot

In this demo I will open Visual Studio Code. There is also a web-based version, so you can access your projectes from anywhere, but many plugins do not run on the online version. I will go through the different panes. I will explain what Markdown is. For this i will open the website: https://mdtotext.com/

A demo Product Requirement Document (PRD) or Tasklist is opened. You can call it anything you like, Tasklist, copilot The document will show the dependencies, like style.css, and an images folder.

Then I will explain what Github is, and Github Copilot. I will demonstrate how a task is executed in Github Copilot.

### Slides

5. What else can we do? I used it to analyse and order text files. I used it to visualize a large data set in an interactive web-site. I think I just scratched the surface of what I can do with it.
6. How much does it costs? The software is free to download and has many open plugins. The Github Copilot basis version I think is free. I have a subscription of 10 euros per month that I pay out of my own pocket. The sofware and platforms are from Microsoft, just like our teams and all the other software the RIVM is using.
7. Do you see any other opportunities for implementing this pipeline for other projects? Any other questions from listeners?
